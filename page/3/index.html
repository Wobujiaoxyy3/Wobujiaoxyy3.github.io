Option

在其它编程语言中，往往都有一个null关键字，该关键字用于表明一个变量当前的值为空（不是零值，例如整型的零值是0），也就是不存在值。当你对这些null进行操作时，例如调用一个方法，就会直接抛出null异常，导致程序的崩溃，因此我们在编程时需要格外的小心去处理这些null空值。

尽管如此，空值的表达依然非常有意义，因为空值表示当前时刻变量的值是缺失的。有鉴于此，Rust吸取了众多教训，决定抛弃null，而改为使用Option枚举变量来表述这种结果。

引用和借用

上节中提到，如果仅仅支持通过转移所有权的方式获取一个值，那会让程序变得复杂。 Rust 能否像其它编程语言一样，使用某个变量的指针或者引用呢？答案是可以。

Rust通过借用(Borrowing) 这个概念来达成上述的目的，获取变量的引用（reference），称之为借用(borrowing)。

引用和解引用

常规引用是一个指针类型，指向了对象存储的内存地址，因而可以在不改变所有权的情况下访问一个值。在下面代码中，我们创建一个i32值的引用y，然后使用解引用运算符来解出y所使用的值:

变量x存放了一个i32值5,可以断言x等于 5。然而，y是x的一个引用。如果希望对y的值做出断言，必须使用*y来解出引用所指向的值（也就是解引用）。一旦解引用了y，就可以访问y所指向的整型值并可以与5做比较。

在Rust中，引用分为不可变引用和可变引用：

不可变引用（Immutable Reference）: 通过不可变引用，我们可以读取数据，但不能修改数据。一个变量可以同时拥有多个不可变引用，但不能与可变引用共存。

可变引用（Mutable Reference）: 通过可变引用，我们可以在读取数据的同时修改数据。一个变量只能拥有一个可变引用，且不能与不可变引用共存。

引用的规则

· 同一时间内，一个变量只能有一个可变引用或多个不可变引用
· 引用必须总是有效

Rust是一种系统编程语言，它的设计目的是确保内存安全并防止数据竞争，而不去依赖GC（垃圾回收器），因为GC会带来性能、内存占用以及Stop the world等问题。我们将在这一讲中介绍Rust的所有权系统。

·为什么所有权系统能够实现零开销内存回收？

内存回收方式对比

C/C++
·内存管理方式：手动管理（Manual Management），手动分配和释放内存，没有内置的垃圾回收系统
·优点：高效且灵活，适用于对性能要求极高的系统级编程
·缺点：容易出现内存泄漏、悬垂指针和缓冲区溢出等问题，需要非常小心的管理内存

JavaScript
·内存管理方式：垃圾回收机制（Garbage Collection）
·特点：浏览器和Node.js环境中均使用垃圾回收器；采用标记-清除（Mark-and-Sweep）、标记-压缩（Mark-and-Compact）、分代回收等算法
·优点：自动内存管理，适合快速开发和运行在多平台上的应用
·缺点：垃圾回收机制需要定时回收内存，在某些情况下可能会导致性能问题，如UI线程停顿等

Rust
·内存管理方式：所有权系统（Ownership System）
·特点：Rust使用所有权系统进行内存管理，编译器在编译时通过静态分析来确保内存的安全
每个值都在任意时刻都有且只有一个有效的所有者
通过借用（引用）机制来共享数据，同时确保数据竞争和悬垂指针的安全
·优点：在编译时就保证内存安全，运行时零开销；避免了数据竞争和悬垂指针
·缺点：需要程序员理解和遵循所有权和借用规则，学习曲线陡峭

栈（Stack）和堆（Heap）

栈和堆事编程语言中最为核心的数据结构。对于Rust这样的系统编程语言来说，值位于栈上还是堆上十分重要，这回影响程序的行为和性能

栈和堆的核心目标就是为程序在运行时提供可供使用的内存空间

栈（Stack）

栈是一种有组织的数据结构，它从低地址到高地址连续的储存数据，同时在需要取出值时，以相反的顺序从高地址到低地址取出值

增加数据叫做进栈，移出数据叫做出栈

因为上述的实现方式，栈中的所有数据都必须占用已知且固定大小的内存空间，假设数据大小是未知的，那么在取出数据时，你将无法取到你想要的数据

堆（Heap）

与栈不同，对于大小未知或者可能变化的数据，我们需要将它存储在堆上（因而堆是一种相对缺乏组织的数据结构）

当向堆上放入数据时，需要请求一定大小的内存空间。操作系统在堆的某处找到一块足够大的空位，把它标记为已使用，并返回一个表示该位置地址的指针, 该过程被称为在堆上分配内存，有时简称为 “分配”(allocating)

接着，该指针会被推入栈中，因为指针的大小是已知且固定的，在后续使用过程中，你将通过栈中的指针，来获取数据在堆上的实际内存位置，进而访问该数据

性能区别

在栈上分配内存比在堆上分配内存要快，因为入栈时操作系统无需进行函数调用（或更慢的系统调用）来分配新的空间，只需要将新数据放入栈顶即可。相比之下，在堆上分配内存则需要更多的工作，这是因为操作系统必须首先找到一块足够存放数据的内存空间，接着做一些记录为下一次分配做准备，如果当前进程分配的内存页不足时，还需要进行系统调用来申请更多内存。 因此，处理器在栈上分配数据会比在堆上分配数据更加高效。

所有权与堆栈

当你的代码调用一个函数时，传递给函数的参数（包括可能指向堆上数据的指针和函数的局部变量）依次被压入栈中，当函数调用结束时，这些值将被从栈中按照相反的顺序依次移除。

因为堆上的数据缺乏组织，因此跟踪这些数据何时分配和释放是非常重要的，否则堆上的数据将产生内存泄漏 —— 这些数据将永远无法被回收。这就是 Rust 所有权系统为我们提供的强大保障。

所有权原则

Rust的所有权系统有三个基本原则：

  1. Rust中每一个值都被一个变量所拥有，该变量被称为值的所有者（Owner）
  2. 在任意时刻，一个值只能有一个所有者
  3. 当所有者离开作用域（Scope）时，该值就会被丢弃

Rust 是一种系统级编程语言，专注于速度、内存安全性和并发性。它由 Mozilla Research 开发，并逐渐成为开源社区中的一个重要成员。Rust 设计用于构建可靠且高效的软件，尤其适用于需要高性能和资源控制的应用场景，如操作系统、游戏引擎、浏览器组件、文件系统和其他需要直接访问硬件的情况。
